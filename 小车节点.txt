C区域自动识别于跑图
	固定地图，固定路线，固定识别点
	先在地图里记录已知点坐标
		停车点
		二维码点
		人像识别点们
		C 区入口点
		C 区顺时针/逆时针关键路点
	这些点都用统一 map 坐标系保存。
	比赛开始时给小车一个初始坐标
	例如停车点就是 (x0, y0, yaw0)
	告诉系统小车现在在地图里的这个位置
	运行中用里程计持续更新当前位置
	根据轮速 / IMU / 里程计积分
	得到一个实时估计位姿
	每到已知固定点时做一次校正
	比如到二维码点
	或到某个人像识别点
	如果你能确认“现在就是这个固定点”，那就把当前位姿重新拉回该点坐标附近
	二维码结束后切到 C 区固定路线
	根据二维码结果选择 cw 或 ccw
	然后一路按 C 区的关键点跑
	中间遇到障碍物时做局部避障
	到识别点再停车识别


第 1 步：确认 map.png 能否作为参考地图
先确认它是不是严格按 500x500 cm -> 500x500 px 等比例画的。
再确认关键尺寸是否对得上，比如通道宽、诊疗室尺寸、二维码点位置。
这里的目标不是马上让 ROS 用它导航，而是先确认它能不能做统一 map 坐标参考。
第 2 步：先定义 map 坐标系和关键点
定义左下角为 (0, 0)，单位米。
把这些点先标出来：
停车点
二维码点
C 区入口点
C 区出口点
cw 路线点
ccw 路线点
这一步很重要，因为后面所有控制都依赖这些点。
第 3 步：先做“当前位置估计”
这一步比“加导航节点”更优先。
至少要有：
初始位姿
里程计累计
到固定锚点时校正
不然即使你有地图和导航点，小车也不知道自己现在在哪，没法追点。
第 4 步：做固定航点执行
先别急着上全套复杂导航。
先做一个最小版本：
给一组点
让车按顺序追
到点切下一个
先验证“无障碍情况下能不能稳定跑完 C 区一圈”。
第 5 步：再做 cw / ccw 两套路点
这时候只需要两份列表：
route_cw.yaml
route_ccw.yaml
二维码结束后，根据结果选一条即可。
第 6 步：把避障接进去
这里不要做成“遇障碍就重新规划整条路”。
更适合你的是：
当前仍有一个目标点
遇到障碍物先局部避开
避开后继续回到当前目标点或下一个目标点
也就是“路线主导，避障补充”。
第 7 步：最后做“导航模式 -> 跑线模式”的切换
比如在 C 区出口或回到某个接线点后：
停掉固定航点执行
恢复 racing_control + racing_track_detection
再沿线回停车点
这一步本质上是两个控制模式的切换。
