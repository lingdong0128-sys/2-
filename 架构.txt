智行锐进 1 队方案架构说明（赛前备案版）

一、队伍与平台信息
参赛队伍基本信息：智行锐进1队-杭州职业技术大学-吴陈熙儿
RDK 型号：X5

二、总体技术路线
1 队采用“统一任务启动 + 固定地图定位 + 二维码触发路线切换 + C 区航点执行 + 巡线收尾”的总体思路。
比赛主流程为：巡线 -> 扫码 -> 跑航点 -> 巡线，全程保留障碍物检测与局部避障能力。
在地图使用上，系统以统一 map 坐标系管理停车点、二维码点、人像识别点、C 区入口点以及顺时针/逆时针关键路点，并在任务开始时给定初始位姿。

三、系统分层结构
1. 基础运行层
底盘节点通过 `ros2 launch origincar_base origincar_bringup.launch.py` 启动。
MCP 通信节点通过 `ros2 launch rosbridge_server rosbridge_websocket_launch.xml` 启动。
深度相机通过 `ros2 launch deptrum-ros-driver-aurora930 aurora930_launch.py` 启动。
双目相机通过 `ros2 launch robot_camera_bringup dual_camera.launch.py` 启动。
上述基础节点由 `robot-startup` 服务负责开机拉起，保障底盘、通信和视觉输入先完成就绪。

2. 任务编排层
比赛版本采用统一总控入口组织任务链路，外层由 `race_start.launch.py` 拉起 `bringup_real.launch.py`，并在入口侧按需启动避障节点。
其中 `bringup_real.launch.py` 对外提供 `use_sim_time`、`autostart`、`log_level`、`launch_obstacle_detection`、`enable_fixed_start_reset` 五个关键参数，用于控制时钟源、生命周期自动激活、日志等级、避障节点和固定起点复位策略。

3. 定位与导航层
该层负责加载固定地图、启动 map server 及其生命周期管理节点，并完成坐标对齐与任务执行前的位姿准备。
建议在地图起点场景下启用 `enable_fixed_start_reset=true`，在任务开始时执行一次固定起点归零，保证小车初始位姿与地图参考一致。
运行过程中系统持续融合里程计、轮速和 IMU 信息估计当前位姿，并在二维码点、识别点等固定锚点位置进行校正。

4. 感知与任务执行层
二维码识别结果用于在 `route_cw.yaml` 和 `route_ccw.yaml` 两套 C 区路线之间进行切换。
进入 C 区后，小车以固定航点顺序执行为主，中间遇到障碍物时采用“路线主导、局部避障补充”的策略，不重新规划整张路线。
到达人像识别点或关键任务点后执行停车识别，完成后继续恢复航点执行。

5. 模式切换层
在完成 C 区固定路线后，系统从跑图模式切回巡线模式，恢复 `racing_control + racing_track_detection` 的收尾链路，最终返回停车点。

四、1 队方案特点
1 队更强调“固定地图下的任务确定性执行”。
技术重点放在已知关键点建图、起点归零、二维码分流、固定航点追踪和锚点校正上，使比赛流程可控、可复盘、便于赛前备案和赛后优化。


智行锐进 2 队方案架构说明（赛前备案版）

一、队伍与平台信息
参赛队伍基本信息：智行锐进2队-杭州职业技术大学-吴鸿轩
RDK 型号：X5

二、总体技术路线
2 队采用“感知先行、状态估计驱动、任务分阶段切换”的实现框架。
整体流程同样覆盖巡线、扫码、C 区跑图和回归巡线四个阶段，但描述上更强调模块协同与状态切换，而不是单纯的固定点追踪。
系统将地图坐标、任务阶段、二维码结果和关键识别点统一纳入任务调度逻辑，使每一段动作都可挂接到明确的状态节点上。

三、系统架构划分
1. 设备接入层
底盘、rosbridge、深度相机和双目相机作为最底层输入输出链路，由 `robot-startup` 在开机后统一拉起。
该层的目标是先保证运动控制、外部通信以及视觉数据源稳定在线，为上层任务提供连续输入。

2. 状态估计层
系统在比赛开始时先注入停车点初始位姿，再依赖里程计、轮速和 IMU 持续更新当前位置。
当车辆经过二维码点、人像识别点或其他可确认的固定位置时，系统将当前位置重新拉回到地图中的标准参考点附近，从而减小累计误差。

3. 任务调度层
2 队仍然通过统一 launch 链路组织比赛节点，但更强调“按阶段推进”的调度思想。
`race_start.launch.py` 负责触发整体流程，`bringup_real.launch.py` 负责承接核心运行节点，并通过 `use_sim_time`、`autostart`、`log_level`、`launch_obstacle_detection`、`enable_fixed_start_reset` 完成运行期开关控制。
其中 `enable_fixed_start_reset` 作为起跑阶段的关键控制项，只建议在固定起点使用。

4. 路线执行层
二维码识别结束后，系统不会临时生成整张新路径，而是在既定路线资源中选择顺时针或逆时针分支。
车辆进入 C 区后按关键航点推进，局部障碍出现时仅对当前目标点附近进行短距离绕行，障碍清除后继续回到原任务链。
这种方式更突出“状态机切换 + 固定路线执行”的协同关系。

5. 收尾闭环层
在 C 区出口或预设接线点完成任务后，系统停掉固定航点执行逻辑，再切回巡线控制链路，完成末段回库或回停车点动作。

四、2 队方案特点
2 队更强调“模块协同和状态驱动”。
与 1 队相比，2 队的话术重点不放在固定航点本身，而放在设备接入、位姿估计、阶段切换、路线分支选择和任务闭环控制，便于体现另一种工程组织思路。
